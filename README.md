# TestPlanwithAI

This repository serves as an experimental workspace for generating Quality Assurance (QA) Test Plans and Test Cases using various Artificial Intelligence (AI) tools and Large Language Models (LLMs). The primary focus of these experiments is the VWO Login Dashboard application (app.vwo.com), using its Product Requirements Document (PRD) as the baseline for test generation.

## 🚀 Project Purpose

The goal of this project is to explore, compare, and demonstrate how different AI coding assistants and LLMs can be leveraged to automate and enhance the QA documentation process. By providing the same PRD and context to different AI tools, we can evaluate their effectiveness in generating comprehensive, structured, and accurate test plans and executable test cases.

## 📁 Repository Structure

The repository is organized by the AI tool or approach used for test generation:

### `VS_GithubCopilot/`
Contains test plan generators and outputs created using GitHub Copilot.
- **`generate_vwo_testplan_xlsx.py`**: A Python script that generates a structured Excel test-plan workbook (`VWO_Login_TestPlan.xlsx`), including sections for Strategy, Scope, and Missing Information.
- **`Enhance_Test_Prompt`**: Prompts and context used to guide the AI in generating test requirements.
- **Outputs**: Generated `.xlsx` and `.md` test plan files.

### `ollama/`
Contains test case generation experiments using local LLMs via Ollama.
- **`Enhanced_Test_Prompt`**: The specific prompt engineering used to query the local LLM for test cases.
- **`ollama_testcase.xlsx`**: The resulting test cases generated by the Ollama model.

### Root Directory
- **`Product Requirements Document_ VWO Login Dashboard.pdf`**: The foundational PRD used as the source of truth for all AI test generation experiments across the different tools.

## 🛠️ Usage

Navigate to the specific tool directory (e.g., `VS_GithubCopilot`) to find instructions and scripts for running that particular test generation approach. Each directory contains its own artifacts and potentially its own `README.md` with detailed execution steps.

## Requirements

Dependencies vary based on the specific generation script being run. 
- Python scripts generally require Python 3.8+ and libraries like `openpyxl`.
- Local LLM generation requires [Ollama](https://ollama.com/) to be installed and running.
